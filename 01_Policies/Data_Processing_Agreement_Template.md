## 1. Definitions

**"Personal Data"** means any information relating to an identified or identifiable natural person.

**"Processing"** means any operation performed on Personal Data (collection, storage, use, disclosure, deletion).

**"Data Subject"** means the individual whose Personal Data is processed.

**"Sub-processor"** means any third party engaged by Processor to process Personal Data.

## 3. Processor Obligations

### 3.1 Processing Instructions

Processor shall:
- Process Personal Data only on documented instructions from Controller
- Inform Controller if instructions infringe applicable data protection law
- Not process data for any other purpose without prior written consent

### 3.2 Confidentiality

Processor shall:
- Ensure personnel are bound by confidentiality obligations
- Limit access to authorized personnel only
- Apply need-to-know access principles

### 3.3 Security Measures

Processor shall implement appropriate technical and organizational measures including:

**Per EcoMetricx Encryption Standards:**
- Encryption at rest (AES-256 minimum)
- Encryption in transit (TLS 1.2+)
- Multi-factor authentication for access
- Regular security assessments

**Per EcoMetricx Access Control Policy:**
- Role-based access control
- Least privilege access
- Regular access reviews
- Audit logging

### 3.4 Sub-processing

Processor shall:
- Not engage Sub-processors without prior written authorization
- Ensure Sub-processors are bound by equivalent obligations
- Remain liable for Sub-processor acts
- Maintain list of Sub-processors and notify of changes

**Approved Sub-processors:**
1. [Sub-processor name] - [Purpose]
2. [Sub-processor name] - [Purpose]

### 3.5 Data Subject Rights

Processor shall assist Controller with:
- Access requests (respond within 30 days)
- Rectification requests
- Erasure requests ("right to be forgotten")
- Data portability requests
- Objection to processing

### 3.6 Data Breach Notification

Processor shall:
- Notify Controller within **24 hours** of becoming aware of a data breach
- Provide sufficient information to meet regulatory notification requirements
- Cooperate with breach investigation and remediation
- Document all breaches and actions taken

### 3.7 Audit Rights

Controller may:
- Request evidence of compliance (certifications, audit reports)
- Conduct audits with reasonable notice (30 days minimum)
- Require remediation of findings within agreed timeframes

## 4. Controller Obligations

Controller shall:
- Provide documented processing instructions
- Ensure lawful basis for processing
- Respond to data subject requests
- Notify Processor of any changes affecting processing

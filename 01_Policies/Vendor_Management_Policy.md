# Vendor Management Policy

**Version:** 1.0  
**Effective Date:** December 2025  
**Owner:** EcoMetricx InfoSec Team (Andrei Ionete - andrei@ecometricx.com)  
**Review Cycle:** Annually or following major organizational or regulatory change

## 1. Purpose

This policy establishes requirements for managing third-party vendors and service providers to ensure they meet EcoMetricx's security, privacy, and compliance standards. This policy supports SOC 2 Trust Service Criteria CC3.2 (Vendor Management).

## 2. Scope

This policy applies to:
- All third-party vendors and service providers
- All contractors and consultants
- All cloud service providers
- All software-as-a-service (SaaS) providers
- All vendors with access to EcoMetricx data or systems

## 3. Vendor Classification

### 3.1 Critical Vendors

- Access to Confidential or Restricted data
- Access to production systems
- Business-critical services
- Regulatory compliance requirements
- **Assessment:** Comprehensive security assessment required

### 3.2 Standard Vendors

- Limited access to Internal data
- Non-production system access
- Standard business services
- **Assessment:** Standard security assessment

### 3.3 Low-Risk Vendors

- Public data only
- No system access
- Commodity services
- **Assessment:** Basic security questionnaire

## 4. Vendor Selection and Onboarding

### 4.1 Vendor Selection Process

1. Business need identified
2. Vendor security assessment
3. Risk assessment
4. Management approval
5. Contract negotiation
6. Security requirements included in contract

### 4.2 Security Assessment

**Assessment Components:**
- Security questionnaire
- SOC 2 or ISO 27001 certification review
- Security audit reports
- Incident response capabilities
- Data protection measures
- Compliance certifications

**Assessment Criteria:**
- Information security program
- Access controls
- Encryption capabilities
- Incident response procedures
- Business continuity planning
- Compliance with applicable regulations

### 4.3 Contract Requirements

**Security Requirements:**
- Security controls and measures
- Data protection requirements
- Access control requirements
- Incident notification requirements
- Audit and assessment rights
- Data breach notification (24-72 hours)
- Right to audit
- Data ownership and return
- Termination and data deletion

**Compliance Requirements:**
- SOC 2 Type II certification (for critical vendors)
- GDPR compliance (if applicable)
- HIPAA compliance (if applicable)
- PCI-DSS compliance (if applicable)
- Other applicable regulations

## 5. Vendor Access Management

### 5.1 Access Requirements

- Vendor-specific accounts (no shared accounts)
- Multi-factor authentication required
- Least privilege access
- Time-limited access when possible
- Regular access reviews

### 5.2 Access Approval

- Business justification required
- Data owner approval
- Security team approval
- Management approval for critical access

### 5.3 Access Monitoring

- All vendor access logged
- Regular access reviews (quarterly)
- Monitoring for unusual activity
- Immediate revocation upon contract termination

## 6. Ongoing Vendor Management

### 6.1 Regular Reviews

- **Critical Vendors:** Annual security assessment
- **Standard Vendors:** Biennial security assessment
- **All Vendors:** Quarterly access reviews
- **All Vendors:** Annual contract review

### 6.2 Security Monitoring

- Monitor vendor security incidents
- Review vendor security updates
- Assess vendor security posture changes
- Review vendor compliance status

### 6.3 Performance Monitoring

- Service level agreement (SLA) monitoring
- Security incident tracking
- Compliance status monitoring
- Contract compliance verification

## 7. Vendor Incident Response

### 7.1 Vendor Security Incidents

- Vendor must notify within 24-72 hours of security incident
- Assess impact on EcoMetricx
- Coordinate response if needed
- Document incident and response

### 7.2 Data Breach Notification

- Vendor must notify within 24-72 hours of data breach
- Assess data exposure
- Notify affected parties per requirements
- Regulatory notification if required

## 8. Vendor Offboarding

### 8.1 Termination Procedures

1. Notify vendor of termination
2. Revoke all access immediately
3. Retrieve all EcoMetricx data
4. Verify data deletion from vendor systems
5. Obtain certificate of data deletion
6. Update vendor records

### 8.2 Data Return and Deletion

- All data returned in secure format
- All data deleted from vendor systems
- Certificate of deletion obtained
- Verification of deletion completed

## 9. Cloud Service Providers

### 9.1 AWS and Cloud Providers

- Security assessment of cloud provider
- Review of cloud provider certifications (SOC 2, ISO 27001)
- Shared responsibility model understood
- Cloud security controls implemented
- Regular security reviews

### 9.2 SaaS Providers

- Security assessment
- Data residency requirements
- Data encryption requirements
- Access control requirements
- Compliance certifications

## 10. Compliance Requirements

### 10.1 SOC 2 CC3.2

- Vendor management program established
- Vendor security assessments conducted
- Vendor contracts include security requirements
- Ongoing vendor monitoring

### 10.2 Regulatory Requirements

- **GDPR:** Data processing agreements with vendors
- **HIPAA:** Business associate agreements (BAAs)
- **PCI-DSS:** Vendor compliance requirements
- **Other Regulations:** As applicable

## 11. Vendor Risk Management

### 11.1 Risk Assessment

- Assess vendor risk during selection
- Ongoing risk assessment
- Risk-based vendor management
- Risk mitigation strategies

### 11.2 Risk Mitigation

- Security requirements in contracts
- Regular security assessments
- Access controls and monitoring
- Incident response coordination
- Business continuity planning

## 12. Documentation and Records

### 12.1 Vendor Records

- Vendor inventory
- Security assessments
- Contracts and agreements
- Access records
- Incident reports
- Review documentation

### 12.2 Record Retention

- Vendor records retained per retention policy
- Security assessments: 7 years
- Contracts: Duration of contract + 7 years
- Incident reports: 7 years

## 13. Responsibilities

### 13.1 Procurement/Vendor Management

- Vendor selection and onboarding
- Contract management
- Vendor relationship management
- Performance monitoring

### 13.2 Information Security Team

- Security assessments
- Security requirement definition
- Ongoing security monitoring
- Incident coordination

### 13.3 Data Owners

- Data classification
- Access approval
- Data protection requirements
- Data return/deletion verification

## 14. Training

- Vendor management training for relevant personnel
- Security assessment training
- Contract negotiation training
- Annual refresher training

## 15. Policy Violations

Non-compliance may result in:
- Vendor contract termination
- Access revocation
- Legal action
- Disciplinary action for employees
